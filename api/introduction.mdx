<Note>
soc2doc does not expose a public API yet. This section describes the direction of
the developer surface being considered. No endpoint on these pages is callable, and
no request or response example here is real.
</Note>

## Why this section exists

It is better to document the boundary clearly than to imply an API exists before it
does. When a public API ships, this section will be tied to a real specification
and updated with verifiable behavior.

Until then, treat this as product direction only.

## The shape we are considering

The first developer capabilities will likely follow the product workflow:

- **Submit a report for review** - start the same Commitment Review flow available
  today, including NDA and authorization requirements.
- **Retrieve commitments** - access the reviewed commitment and cadence output in a
  structured form.
- **Check review status** - see where a submitted report sits in extraction and
  practitioner verification.

<Warning>
This list is direction, not a contract. Field names, resources, and behavior are
not finalized and do not exist until the API is released. Do not build against
anything here yet.
</Warning>
