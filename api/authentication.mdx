## Status

The authentication model for the developer preview is still being decided. When the
preview opens, this page will document exactly how access is granted and how
requests are authorized — with real, verifiable steps and no invented tokens.

What we can say now, honestly:

- Access will be **request-gated** through the developer preview, not open by
  default.
- Credentials will be **scoped to your own organization's data** — the same
  need-to-work boundary described in [confidentiality](/product/confidentiality).

<Warning>
Do not attempt to authenticate against any host yet. No endpoint, key format, or
token scheme on this page exists. This section becomes real only when the API is
released and bound to a published specification.
</Warning>
