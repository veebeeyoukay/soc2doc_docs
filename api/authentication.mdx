<Note>
There is no public API to authenticate against today, and no credentials are
issued. Nothing on this page is callable.
</Note>

## Status

The authentication model for the developer preview is still being decided. When the
preview opens, this page will document how access is granted and how requests are
authorized, using real steps tied to a released API.

What we can say now:

- Access will be **request-gated** through the developer preview.
- Credentials will be **scoped to the customer's own data**, matching the
  need-to-work boundary described in [confidentiality](/product/confidentiality).

<Warning>
Do not attempt to authenticate against any soc2doc API host yet. No endpoint, key
format, or token scheme on this page exists.
</Warning>
